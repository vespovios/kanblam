# Self-host

KanBlam is open source under the [AGPL-3.0
license](https://github.com/vespovios/kanblam/blob/main/LICENSE). If you're
comfortable with Docker, you can run your own copy on a server you
control. No accounts, no fees, no opinions about who hosts your data.

## The short version

```bash
git clone https://github.com/vespovios/kanblam.git
cd kanblam
cp .env.example .env.prod

# Generate three secrets and fill them in .env.prod:
echo "NEXTAUTH_SECRET=$(openssl rand -base64 32)"
echo "CRON_SECRET=$(openssl rand -base64 32)"
echo "POSTGRES_PASSWORD=$(openssl rand -hex 24)"

# Edit .env.prod — at minimum set:
#   LANDING_MODE=app
#   WAITLIST_ENABLED=false
#   NEXTAUTH_URL=https://your-domain
#   APP_URL=https://your-domain
#   DATABASE_URL inlining the POSTGRES_PASSWORD value
#   ADMIN_EMAIL / ADMIN_PASSWORD for the seed admin account
chmod 600 .env.prod

docker compose -f docker/docker-compose.prod.yml --env-file .env.prod up -d --build
```

Browse to your domain, log in with `ADMIN_EMAIL` /
`ADMIN_PASSWORD`, change the password from Settings, you're live.

## Feature flags

The same codebase serves both the public hosted service at
kanblam.com and self-host deployments. A small set of env vars
controls hosted-only behaviour — set these in your `.env.prod`:

| Flag | Default (SaaS) | Self-host value | What it does |
|---|---|---|---|
| `LANDING_MODE` | `marketing` | `app` | Skip the marketing landing page at `/`; logged-out visitors go to `/login` |
| `WAITLIST_ENABLED` | `true` | `false` | Disable `/api/waitlist` (returns 404) and hide the waitlist form |
| `BILLING_ENABLED` | `false` | `false` | Reserved for future Polar.sh billing — has no effect today |

## Updates

```bash
cd /path/to/kanblam
git pull
docker compose -f docker/docker-compose.prod.yml --env-file .env.prod up -d --build
```

Database migrations run automatically on container start. Your
Postgres data is preserved on the named volume.
